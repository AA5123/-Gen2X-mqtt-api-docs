Impinj Gen2X MQTT API Tutorial

IoT Connector MQTT Interface
Applicable Devices: Zebra Fixed RFID Readers (FXR90, FX9600, ATR7000)

Overview

This tutorial provides a walkthrough of the steps to use Impinj Gen2X tag features via the MQTT API on Zebra fixed RFID readers, including:

  • Protected Mode (Tag Protection)
  • TagFocus
  • Tag Quieting
  • FastID

All Gen2X commands are sent as JSON payloads to the MQTT command topic and responses are received on the response topic. Each command uses the same top-level structure:

{
  "command": "set_impinjGen2X",
  "command_id": "<uuid>",
  "payload": { ... }
}

Supported Features

Tag Protection (Protected Mode)

Impinj TagProtect prevents unauthorized cloning and reading of tags using a password-based protection mechanism. It renders a tag invisible to an RFID reader unless the correct 32-bit password is provided.

CommandDescription
enableTagProtectionLocks a specific tag so no reader can see it without the correct password
disableTagProtectionRemoves the lock — the tag becomes visible to all readers again
enableTagVisibilityTells the reader "here is the password, now show me protected tags during scanning"
disableTagVisibilityRevokes the reader's ability to see protected tags
enableShortRangeProtects the tag AND restricts it to respond only at close proximity

Example — Enable Tag Protection:

{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagProtect": {
      "action": "enableTagProtection",
      "password": "77777777",
      "tagID": "12348765a5050076c4d75211"
    }
  }
}

FastID

Impinj FastID enables tags to return both their EPC and TID (Tag Identifier) in a single inventory response, eliminating the need for a separate TID read operation.

{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "fastID": {
      "enabled": true
    }
  }
}

TagFocus

Impinj TagFocus instructs tags that have already been inventoried to remain unresponsive, enabling the reader to focus on finding unread tags. This is ideal for dense tag environments.

{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagFocus": {
      "enabled": true
    }
  }
}

Tag Quieting

Impinj TagQuieting selectively silences individual tags so they do not participate in inventory rounds. Other tags are unaffected.

{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagQuieting": {
      "basic": {
        "action": "quiet",
        "tagIDs": ["12348765a5050076c4d75211"]
      }
    }
  }
}

How It Works

  1. Configure — Send set_impinjGen2X commands to stage one or more Gen2X features
  2. Verify — Use get_impinjGen2X to read back the current configuration and confirm your settings
  3. Apply — Stop the radio, then send the start command with applyImpinjGen2X: true
Important: The radio must be stopped before applying Gen2X configuration. Starting with Gen2X active while the radio is already running will return an error.

Configure & Apply

Step 1: Verify Current Configuration

{
  "command": "get_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {}
}

The response payload mirrors the last applied configuration. Returns an empty object if no Gen2X configuration has been set yet.

Step 2: Start Radio with Gen2X

{
  "command": "start",
  "command_id": "auto-generated",
  "payload": {
    "applyImpinjGen2X": true
  }
}
Success Response: All Gen2X commands return the same success response structure: 
{
  "command": "set_impinjGen2X",
  "command_id": "same-as-request",
  "payload": {
    "message": "Success: Gen2X configured. Use applyImpinjGen2X flag in start command to apply features."
  },
  "response": "success"
}

MQTT Topics

TopicPurpose
Command topicSend request payloads (set/get commands)
Response topicReceive command responses (success/failure)
Data topicReceive tag read data (inventory events)

All commands use command, command_id, and payload as the top-level JSON fields. The command_id (UUID) correlates each response back to its original request.
