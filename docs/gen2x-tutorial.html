Impinj Gen2X APIs Tutorial

Impinj Gen2X MQTT API 1.0.0
Applicable Devices: Zebra Fixed RFID Readers (FXR90, FX9600, ATR7000)
MQTT
REST API

Overview

This tutorial provides a walk-through of the steps to use Impinj Gen2X tag features via the MQTT API on Zebra fixed RFID readers, including:

  • Protected Mode
  • TagFocus
  • Tag Quieting
  • FastID

Details

Impinj Gen2X is an enhancement to Gen2's radio and logical layers. Impinj Gen2X tags are required to leverage these features with Gen2X-enabled Zebra RFID Readers.

Users should verify whether their tags support these operations by referring to Impinj Gen2X tag specifications: http://www.impinj.com/Gen2X

All Gen2X commands are sent as JSON payloads to the MQTT command topic and responses are received on the response topic. Each command uses the same top-level structure:

{
  "command": "set_impinjGen2X",
  "command_id": "<uuid>",
  "payload": { ... }
}

The command_id (UUID) correlates each response back to its original request.

Setting it up

Connect to the reader's MQTT broker and subscribe to the response topic and data topic. Then publish commands to the command topic using the JSON structures shown below.

All Gen2X configuration commands use "command": "set_impinjGen2X". After configuring the desired features, stop the radio and send a start command with "applyImpinjGen2X": true to apply the configuration.

NOTE: The radio must be stopped before applying Gen2X configuration. Starting with Gen2X active while the radio is already running will return an error.

Protected Mode

Protected mode is a feature that renders a tag invisible to an RFID reader unless the reader first provides the correct 32-bit PIN/Password. A user can make the tag protected or unprotected using the following commands.

Enable Impinj Protected Mode

Locks a specific tag using Impinj TagProtect so it becomes invisible to all readers unless the correct 32-bit password is provided during the inventory.

Example Request — enableTagProtection 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagProtect": {
      "action": "enableTagProtection",
      "password": "77777777",
      "tagID": "e2801191a5030069073b426d"
    }
  }
}
Schema — tagProtect (Protect/Unprotect)
FieldTypeRequiredDescription
actionstringrequiredenableTagProtection or disableTagProtection
passwordstringrequired8-character hex 32-bit password. Pattern: ^[0-9A-Fa-f]{8}$
tagIDstringrequiredHexadecimal EPC of the target tag
enableShortRangebooleanoptionalEnable short-range protection mode

Disable Impinj Protected Mode

Removes the Impinj TagProtect lock from a previously protected tag. The tag becomes visible to all readers again without any password requirement.

Example Request — disableTagProtection 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagProtect": {
      "action": "disableTagProtection",
      "password": "77777777",
      "tagID": "e2801191a5030069073b426d"
    }
  }
}

Tag Visibility

Enable Inventory of Protected Tags

Tells the reader the password so it can see and inventory protected tags that would otherwise be invisible.

Example Request — enableTagVisibility 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagProtect": {
      "action": "enableTagVisibility",
      "password": "77777777"
    }
  }
}
Schema — tagProtect (Visibility)
FieldTypeRequiredDescription
actionstringrequiredenableTagVisibility or disableTagVisibility
passwordstringrequired8-character hex 32-bit password
enableShortRangebooleanoptionalEnable short-range protection mode
Note: tagID must be omitted for visibility actions.

Clear Protected Mode Configuration

Revokes the reader's stored password so it will no longer be able to see protected tags during inventory.

Example Request — disableTagVisibility 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagProtect": {
      "action": "disableTagVisibility",
      "password": "77777777"
    }
  }
}

Short Range

Enable Short Range

Protects a tag AND restricts it to respond only when the reader antenna is within close proximity.

Example Request — enableShortRange 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagProtect": {
      "action": "enableTagProtection",
      "password": "77777777",
      "tagID": "e2801191a5030069073b426d",
      "enableShortRange": true
    }
  }
}

TagFocus

TagFocus allows readers to silence already-inventoried tags to reduce redundant reads, allowing a reader to focus on weak tags that are typically the last to be found.

Example Request — Enable TagFocus 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagFocus": {
      "enabled": true
    }
  }
}
Example Request — Disable TagFocus 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagFocus": {
      "enabled": false
    }
  }
}
Schema — tagFocus
FieldTypeRequiredDescription
enabledbooleanrequiredtrue to enable, false to disable. Targets tags in session S1.

}
Example Request — Unquiet Tags

  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "tagQuieting": {
      "basic": {
        "action": "unquiet",
        "tagIDs": ["e2801191a5030069073b426d"]
      }
    }
  }
}
Schema — tagQuieting.basic
FieldTypeRequiredDescription

tagIDsarrayrequiredNon-empty list of hexadecimal EPC strings. Pattern: ^[0-9A-Fa-f]+$

FastID

FastID enables Impinj tags to backscatter their EPC and TID in a single response for inventory, eliminating the need for a separate TID read operation.

Example Request — Enable FastID 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "fastID": {
      "enabled": true
    }
  }
}
Example Request — Enable FastID with TID Selector 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "fastID": {
      "enabled": true,
      "tidSelector": "TID[0]"
    }
  }
}
Example Request — Disable FastID 
{
  "command": "set_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {
    "fastID": {
      "enabled": false
    }
  }
}
Schema — fastID
FieldTypeRequiredDescription
enabledbooleanrequiredtrue to enable, false to disable
tidSelectorstringoptionalTID word selector: TID[0], TID[1], TID[2], or TID[3]

Get Configuration

Retrieves the current Gen2X configuration from the reader. Returns an empty object if no Gen2X configuration has been set yet.

Example Request 
{
  "command": "get_impinjGen2X",
  "command_id": "auto-generated",
  "payload": {}
}
Example Response (all features configured) 
{
  "fastID": { "enabled": true, "tidSelector": "TID[0]" },
  "tagProtect": {
    "action": "enableTagProtection",
    "password": "77777777",
    "tagID": "e2801191a5030069073b426d",
    "enableShortRange": true
  },
  "tagFocus": { "enabled": true },
  "tagQuieting": {
    "basic": { "action": "quiet", "tagIDs": ["e2801191a5030069073b426d"] }
  }
}

Start with Gen2X

After configuring the desired Gen2X features, stop the radio and then send a start command with applyImpinjGen2X: true.

Example Request 
{
  "command": "start",
  "command_id": "auto-generated",
  "payload": {
    "applyImpinjGen2X": true
  }
}
Success Response 
{
  "command": "set_impinjGen2X",
  "command_id": "same-as-request",
  "payload": {
    "message": "Success: Gen2X configured."
  },
  "response": "success"
}
Important: The radio must be stopped before applying Gen2X configuration. Starting with Gen2X active while the radio is already running will return an error.

Overview

This tutorial provides a walk-through of the steps to use Impinj Gen2X tag features via the IoT Connector REST API on Zebra fixed RFID readers, including:

  • Protected Mode (TagProtect)
  • TagFocus
  • Tag Quieting
  • FastID

Details

Impinj Gen2X is an enhancement to Gen2's radio and logical layers. Impinj Gen2X tags are required to leverage these features with Gen2X-enabled Zebra RFID Readers.

Users should verify whether their tags support these operations by referring to Impinj Gen2X tag specifications: http://www.impinj.com/Gen2X

The REST API uses a single endpoint /cloud/impinjGen2X with two methods:

MethodEndpointDescription
GET/cloud/impinjGen2XRetrieve current Gen2X configuration
PUT/cloud/impinjGen2XSet Gen2X configuration (at least one feature object required)

Authentication

The REST API requires Bearer token authentication. To obtain a token:

  1. If a bearer token was previously copied from the dashboard page, use it directly in the Authorization header.
  2. If not, use Admin login credentials with Basic Auth, execute POST /cloud/localRestLogin, and copy the bearer token from the response.

Include the token in every request:

Authorization: Bearer <your-token>

Setting it up

All requests are sent to the reader's IP over HTTPS (default port 443). Configuration is done via PUT /cloud/impinjGen2X with a JSON body containing at least one feature object (fastID, tagProtect, tagFocus, or tagQuieting).

After configuring the desired features, use the start command with the applyImpinjGen2X flag to begin Gen2X operations.

NOTE: The radio must be stopped before applying Gen2X configuration. The start command with applyImpinjGen2X flag should be used to start Gen2X operations.

Protected Mode

Impinj TagProtect prevents unauthorized cloning and reading of tags using a password-based protection mechanism. It renders a tag invisible to an RFID reader unless the reader first provides the correct 32-bit password.

The required fields depend on the action: tagID is required for enableTagProtection/disableTagProtection and must be omitted for enableTagVisibility/disableTagVisibility.

Enable Tag Protection

HTTP Request 
PUT /cloud/impinjGen2X HTTP/1.1
Host: <reader-ip>
Content-Type: application/json
Authorization: Bearer <token>

{
  "tagProtect": {
    "action": "enableTagProtection",
    "password": "77777777",
    "tagID": "e2801191a5030069073b426d",
    "enableShortRange": true
  }
}
cURL Example 
curl -X PUT https://<reader-ip>/cloud/impinjGen2X \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "tagProtect": {
      "action": "enableTagProtection",
      "password": "77777777",
      "tagID": "e2801191a5030069073b426d",
      "enableShortRange": true
    }
  }'
Schema — tagProtect (Protect/Unprotect)
FieldTypeRequiredDescription
actionstringrequiredenableTagProtection or disableTagProtection
passwordstringrequired8-character hex 32-bit password. Pattern: ^[0-9A-Fa-f]{8}$
tagIDstringrequiredHexadecimal EPC of the target tag
enableShortRangebooleanoptionalEnable short-range protection mode
